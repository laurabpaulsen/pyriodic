class pyriodic.circular.Circular(data: ndarray, labels: list | ndarray | None = None, unit: str = 'radians', full_range: int | None = None)#

Bases: object

A class for representing and working with circular data (e.g., angles, time-of-day, phase).

This class supports circular statistics and visualization of data that wraps around a fixed range, such as angles (in radians or degrees), phases, hours, or other cyclic measurements. It provides basic unit validation, conversion between degrees and radians, and visualisation tools.

Parameters:

  • data (array-like) ‚Äì A sequence of numerical values representing circular measurements. Values should be in the range appropriate to the specified unit.

  • labels (array-like of str or int, optional) ‚Äì Optional condition labels or identifiers corresponding to each data point.

  • unit (str, optional) ‚Äì The unit of the input data. Must be one of {‚Äúradians‚Äù, ‚Äúdegrees‚Äù}. Default is ‚Äúradians‚Äù. Assumes radian range to be from 0 to 2pi.

  • full_range (int, optional)

pyriodic.permutation.permutation_test_against_null(observed: ndarray, phase_pool: ndarray | Literal['uniform'] = 'uniform', n_null: int = 1000, stat_fun: Callable | None = None, alternative: Literal['greater', 'less', 'two-sided'] = 'greater', rng: Generator | None = None, n_bins: int = 30, verbose: bool = True, return_null_distribution=False, return_null_samples=False) → tuple#

Compare an observed circular phase distribution against a null distribution created by random sampling from the underlying phase pool.

Parameters:

  • observed (np.ndarray) ‚Äì 1D array of observed angular data in radians (values should be in [0, 2œÄ]).

  • phase_pool (Union[np.ndarray, Literal["uniform"]]) ‚Äì 1D array of phase values from which to extract null samples or a string ‚Äúuniform‚Äù to use a uniform distribution.

n_nullint, default=1000

Number of null sample-to-null sample comparisons used to generate the null distribution.

stat_funCallable, optional

Function used to compute the test statistic. Must accept at least two 1D arrays. If it accepts an additional n_bins argument, it will be passed automatically. Defaults to watson_u2.

alternativestr, default=‚Äùgreater‚Äù

Defines the alternative hypothesis. Must be one of: - ‚Äúgreater‚Äù: Observed statistic is significantly greater than null. - ‚Äúless‚Äù: Observed statistic is significantly less than null. - ‚Äútwo-sided‚Äù: Observed statistic differs from null in either direction.

rngnp.random.Generator, optional

Random number generator for reproducibility. If None, a default generator is created.

n_binsint, default=30

Number of evaluation points for ECDF-based statistics (used only if stat_fun accepts it).

verbosebool, default=True

If True, prints the observed statistic and corresponding p-value.

Returns:

  • obs_stat (float) ‚Äì Mean test statistic between observed and permuted samples.

  • p_val (float) ‚Äì p-value computed from the null distribution.

  • null (np.ndarray, optional) ‚Äì The null distribution of test statistics from permuted-vs-permuted samples.

  • null_samples (list of np.ndarray, optional) ‚Äì The raw sampled phase vectors used to compute the null distribution.

Notes

This test assumes that the permuted samples represent valid null-distribution surrogates and that the test statistic is sensitive to distributional differences on the circle.

pyriodic.permutation.watson_u2(sample1: ndarray, sample2: ndarray, n_bins: int = 30) → float#

Compute the Watson U¬≤ statistic between two circular samples.

This is a non-parametric test statistic used to compare two samples of circular data, based on the squared differences between their ECDFs.

Parameters: - sample1 (np.ndarray): First sample of circular data (values should be in [0, 2œÄ]). - sample2 (np.ndarray): Second sample of circular data (values should be in [0, 2œÄ]). - n_bins (int): Number of points to evaluate the ECDFs on the circle.

Returns: - float: The Watson U¬≤ statistic.

pyriodic.phase_events.create_phase_events(phase_ts: ndarray, events: ndarray, event_labels: ndarray | None = None, unit: str = 'radians', first_samp: int = 0, rejection_method: str | None = None, rejection_criterion: float | None = None, bad_segments: ndarray | None = None, return_rejected: bool = False) → PhaseEvents | tuple[PhaseEvents, list[int]]#

Create Circular object(s) from a phase angle time series and event markers. Optionally apply cycle-based rejection criteria to exclude events occurring during atypical phase dynamics.

Parameters:

  • phase_ts (np.ndarray) ‚Äì 1D array of phase values (in radians), typically spanning multiple 0‚Äì2œÄ cycles.

  • events (np.ndarray) ‚Äì 1D array of sample indices at which to extract phase values.

  • event_labels (np.ndarray, optional) ‚Äì Labels for grouping events. If None, returns a single Circular object; otherwise, returns a PhaseEvents container grouped by unique labels.

  • unit (str) ‚Äì Unit of the input phase time series and the desired unit for output Circular objects. Must be either ‚Äòradians‚Äô or ‚Äòdegrees‚Äô. Internally, all computations are performed in radians.

  • first_samp (int) ‚Äì Offset to subtract from each event index to align with the phase time series (useful if phase_ts is a segment of a longer recording).

  • rejection_method (str, optional) ‚Äì Method to reject events based on surrounding phase dynamics. Supported values: ‚Äòsegment_duration_sd‚Äô ‚Äî excludes events during rising/falling phase segments whose durations deviate more than rejection_criterion standard deviations from the mean.

  • rejection_criterion (float, optional) ‚Äì Threshold (in standard deviation units) for identifying outlier segments. Only used if rejection_method=‚Äôsegment_duration_sd‚Äô. Default is 3.

  • bad_segments (Optional[np.ndarray]) ‚Äì For ignoring events that fall within specific segments of the phase time series. Shape should be (n_segments, 2), where each row is a (start, stop) index pair in samples.

  • return_rejected (bool) ‚Äì If True, also return a list of rejected event indices.

Returns:

  • Circular or PhaseEvents ‚Äì If event_labels is None, returns a single Circular object. Otherwise, returns a PhaseEvents object containing condition-grouped Circular data.

  • (optional) list[int] ‚Äì If return_rejected is True, returns a second value: the list of event indices that were rejected.
